Pipeline

One thin path from save event to security signal.

The extension remains a client, MCP stays a transport layer, and the API owns the expensive work.

01

Save the file

The editor event triggers a scan the moment code changes.

02

Bridge with MCP

The client sends the request through the local MCP transport.

03

Call the API

The API receives the request and runs security and AI services.

04

Render the result

Diagnostics and panels show findings where developers already work.

05

Track the trend

History and score changes help teams see whether the platform is improving.

Deployment

How to use it like a real extension.

The landing page can ship as static hosting, while the backend and extension are deployed separately.

Local dev

Run the full loop locally

Start the API, load the extension host, and use the demo page to show the product flow without extra setup.

Backend

Deploy the API separately

The API can live on a host like Railway or a container service, while the extension points at `GOAEGIS_API_BASE_URL`.

Landing page

Host this page on static storage

This page is plain HTML, so it can move to GitHub Pages, Netlify, or any static host without a build pipeline.
